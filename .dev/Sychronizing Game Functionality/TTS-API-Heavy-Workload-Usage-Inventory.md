## Scan Method

Catalog rows scanned: 150 (`HEAVY`, `MODERATE`, `LIGHT`, and `CONTEXT`).

Roots scanned:

```powershell
core lib objects ui global .dev/testbed
```

Generated source-adjacent XML under `ui/` was included. `.dev/build-logs/global-bundle-sample.lua` was spot-checked but excluded from the inventory because it duplicates bundled source line numbers. Vendored API docs and prose docs were excluded except where this report explicitly calls out generated XML snippets.

Primary scan:

```powershell
rg --pcre2 --line-number --column --no-heading --glob "*.ttslua" --glob "*.lua" --glob "*.xml" <catalog-symbol-call-pattern> core lib objects ui global .dev/testbed
```

Targeted alias scan for `CONTEXT` component rows:

```powershell
rg -n "\b(comp|component|source|mat|material)\.set\s*\(|\.search\s*\(" core lib objects ui global .dev/testbed --glob "*.ttslua" --glob "*.lua" --glob "*.xml"
```

Important false-positive rules:
- `copy` hits were `Vector:copy()` helpers, not global `copy(object_list)`.
- `split` hits were string/helper `split` functions, not Object deck `split()`.
- `spread` hits were comments about snap ordering, not Object deck `spread()`.
- `group` hits were prose/table names, not global `group(object_list)`.
- `getObjects` rows below distinguish global `getObjects()` from object/container/zone `.getObjects()`.
